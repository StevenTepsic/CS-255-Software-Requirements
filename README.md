# CS-255-Software-Requirements
CS-255 Software Requirements

This System was to provide students with a platform to practice DMV tests, study DMV content, and schedule in person driving lessons.

One area I could have improved was better doccumentation of the technical and nonfunctional requirements in the Buisness Requirements Doc as well as expanding on them in the System Design Doc.

I was able to interpret the user's needs by turning their interview into defined testable system requirements.

I approach designing software by starting with the client's needs first, understanding what the system has to do before thinking about how to build it. Going forward I would continue using tools like use case diagrams, sequence diagrams, and requirements documents to break the system down into clear, manageable pieces before any development starts.
